Bienvenido a la Aplicación de Prueba Técnica

Objetivos del Proyecto

El objetivo principal de esta prueba técnica fue crear una aplicación web sencilla utilizando Angular que consuma datos de una API pública y los presente de manera clara y funcional. Este proyecto tiene como propósito demostrar mi capacidad para trabajar con Angular, manejar peticiones HTTP y diseñar una interfaz de usuario moderna y amigable.

¿Qué hice?

  • Elección de la API Pública:

    Eligi la API de Rick and Morty, una API pública que proporciona información detallada sobre personajes, episodios y ubicaciones de la serie.
    Esta API fue elegida debido a la riqueza de datos y la popularidad del contenido. Tambien por los tiempos de respuesta de la misma y la nula autenticacion para realizar peticiones.

  • Componentes Creados:

    • Modulo Pages: Administra la navegación y la estructura general de la aplicación. Contiene los principales componentes de la app.
    • Modulo de Shared: Dentro de esta seccion de la aplicacion se encuentran componentes que pueden ser usados dentro de toda la app.
      Ejemplos del mismo son los botones con animaciones personalizadas, una tabla dinamica para visualizar la informacion, el componente Dialog
      para visualizar la informacion detallada de los componentes, las vistas para los errores y los loaders.
    • Vista de Table-View: Muestra una lista de personajes con información relevante como nombre, especie y estado en formato de tabla teniendo
      3 filtros disponibles para poder visualizar la informacion.
    • Vista de Cards-View: Muestra una lista de personajes con información relevante como nombre, especie y estado en formato de Cards teniendo
      3 filtros disponibles para poder visualizar la informacion.

  • Consumo de la API:

    Se utilizó el servicio HttpClient de Angular para realizar llamadas HTTP a la API de Rick and Morty, obteniendo datos en formato JSON.

  • Interactividad del Usuario:

    • Búsqueda: Los usuarios pueden buscar personajes específicos mediante un campo de entrada, filtrando los resultados en tiempo real.
    • Filtrado: La aplicación permite filtrar personajes por estado (vivo, muerto, desconocido) y especie, proporcionando una experiencia de usuario dinámica y personalizada.
    • Selección: Al hacer clic en un personaje, se muestra un diálogo con información detallada del mismo.

  • Diseño con Angular Material:

    Se utilizó Angular Material para darle un aspecto moderno y limpio a la aplicación, con componentes como botones, diálogos, tarjetas, y formularios, asegurando una interfaz consistente y atractiva.
    La aplicación está diseñada para ser completamente responsive, garantizando una experiencia de usuario óptima tanto en dispositivos móviles como en pantallas de escritorio.

  • Validación y Manejo de Errores:

    Se implementó una validación básica para el campo de búsqueda, asegurando que las entradas sean manejadas de manera adecuada. Se manejan errores de la API, como la falta de conexión o respuestas no válidas, mostrando mensajes de error claros y concisos para mejorar la experiencia del usuario.
